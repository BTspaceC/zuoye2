## 二、推荐模型提取与 SVD 潜在空间转换

### 1. 长格式评分数据的矩阵转换
本实验所提供的数据文件与实验七不同，`processed_ratings.csv` 为长格式数据（包含 `user_id`、`joke_id` 和 `rating` 三列）。算法在计算相似度前，必须先将其转换为宽格式的“笑话-用户”特征矩阵，并对缺失的评分进行 0 填充。

### 2. SVD 潜在因子空间降维
为了缓解数据稀疏性问题并提取深层兴趣维度，本实验采用奇异值分解（SVD）潜在因子模型进行推荐：
1. **降维压缩**：使用 `scikit-learn` 的 `TruncatedSVD`，将超高维稀疏评分矩阵压缩为稠密的潜在因子矩阵，设定潜在因子维度 $k = 20$；
2. **相似度矩阵构建**：在压缩后的潜在因子空间中，调用 `cosine_similarity` 计算两两笑话之间的余弦相似度。该过程可以有效过滤评分噪声并捕获隐含的语义相关性。

### 3. 个性化加权推荐函数改造
本实验将实验七的“给定单个笑话找相似”改造为“给定用户打分历史进行综合推荐”：
* **输入**：用户对 3 个随机初始笑话的评分向量（数值范围为 $[-10, 10]$）；
* **计算逻辑**：使用用户的打分作为权重，对这 3 个笑话与其他所有候选笑话的相似度向量进行加权求和，计算所有候选笑话的推荐得分：
$$\text{Score}(j) = \sum_{k=1}^{3} R_{k} \times \text{Sim}(j, j_k)$$
* **去重过滤**：为防止重复推荐，计算后需主动将用户已读并打分的这 3 个初始笑话从候选集合中剔除；
* **输出**：排序后推荐得分最高的前 5 个未读笑话。

## 三、Streamlit 推荐应用界面开发

### 1. 界面布局与卡片设计
为了提供流畅的交互，本应用使用了双栏与侧边栏的响应式布局，并通过嵌入 CSS 提升视觉质感：
* **侧边栏（控制台）**：包含学生个人信息配置、SVD 降维模型参数展示（可显示累积解释方差，测试中 $k=20$ 保留了 55.45% 的信息）以及一键重置按钮；
* **初始评测区**：使用 `st.columns(3)` 布局将 3 个随机笑话以高对比度卡片形式并排呈现。底部配备 `st.slider` 滑动条，支持 $[-10.0, 10.0]$ 范围内的浮点数打分；
* **推荐呈现区**：推荐的 5 个笑话以精美的绿色卡片排列。用户可对推荐结果逐一给出反馈评分，用于计算推荐的匹配度；
* **日志与指标区**：计算总体满意度百分比，利用 `st.progress` 展现进度条，并渲染测试日志表。

### 2. 满意度归一化公式
推荐系统效能的核心指标为用户对 5 个推荐笑话的打分满意度。打分范围为 $[-10, 10]$，故总分区间为 $[-50, 50]$。通过以下公式将其归一化到 $[0\%, 100\%]$：
$$\text{Satisfaction}(\%) = \frac{\sum_{i=1}^{5} R^{rec}_i + 50}{100} \times 100\%$$

## 五、云端部署实践

本实验重点在于将本地的 Web 服务部署到公网。此处详细记录两种部署方式的实践细节。

### 方式A：Streamlit Community Cloud 部署

Streamlit Community Cloud 是 Streamlit 官方提供的免费托管服务，它直接与 GitHub 代码库集成，提供自动构建与 CI/CD 部署。

#### 1. 组织项目与 GitHub 托管
在本地将项目整理为扁平的结构，确保入口主文件命名为 `streamlit_app.py`：
```
joke_recommender/
├── Dataset4JokeSet.xlsx             # 笑话文本数据文件
├── processed_ratings.csv            # 预处理评分数据
├── requirements.txt                 # 依赖库清单
├── 实验八报告-黄裕荣-2430130329.md   # 本实验报告文档
└── streamlit_app.py                 # 应用程序入口代码
```
个人 GitHub 仓库链接为：[https://github.com/BTspaceC/zuoye2](https://github.com/BTspaceC/zuoye2)。本地已通过 Git 工具将上述所有文件推送（Git Push）至远程仓库主分支（`main`）。

#### 2. 云端平台关联与构建配置
1. 访问 [share.streamlit.io](https://share.streamlit.io) 登录控制台，选择使用 GitHub 账号授权登录；
2. 点击右上角的 **New App** 按钮；
3. 在部署配置表单中，选择您的 GitHub 仓库名称，Branch 填写 `main`，Main file path 填写 `streamlit_app.py`；
4. 点击右下角的 **Deploy!**。Streamlit Cloud 将自动分配独立的容器沙箱，读取项目中的 `requirements.txt` 并通过 `pip` 安装所需的全部科学计算依赖；
5. 构建完成后，容器自动执行 `streamlit run`，即可生成唯一的公网访问域名（如 `https://[用户名]-[仓库名].streamlit.app`）。

### 方式B：魔搭社区创空间部署

魔搭社区（ModelScope）是国内主流的开源模型社区，其“创空间”提供了便捷的 Streamlit 容器应用托管服务，适合国内网络环境的高速访问。

#### 1. 创建创空间
1. 访问并登录魔搭社区官网（[modelscope.cn](https://www.modelscope.cn)）；
2. 进入“创空间”板块，点击右上角的 **创建创空间** 按钮；
3. 填写基本元数据：
   * **创空间名称**：例如 `svd_joke_recommender`
   * **SDK 类型**：**必须选择 Streamlit**
   * **授权协议**：Apache 2.0
   * **可见性**：公开

#### 2. 上传文件与镜像自动构建
1. 在新建的空间详情页中，选择 **文件** 或 **代码** 标签卡；
2. 使用魔搭网页端的“上传文件”按钮，或使用 Git 命令关联魔搭的远程 Git 地址：
   * 将 `streamlit_app.py`、`processed_ratings.csv`、`Dataset4JokeSet.xlsx` 和 `requirements.txt` 提交并推送到魔搭的 master 分支；
3. 文件推送成功后，魔搭创空间会自动触发镜像构建，读取 `requirements.txt` 完成环境搭建；
4. 状态变为“运行中”后，即可在创空间内以 iframe 形式直接进行笑话推荐系统的在线测试。
